1. Headlines — ALL CAPS to Title Case

Convert each headline below. In Elementor, also remove text-transform: uppercase from the heading's Style panel and retype the text in Title Case so screen readers announce it correctly.

Specific Headlines to Change

  • Hero title: PHONE COMPANIONS→Phone Companions
  • Intro H2: BUILD A FRIENDSHIP FROM THE COMFORT OF YOUR HOME→Build a Friendship from the Comfort of Your Home
  • How It Works step 1: PHONE ASSESSMENT→Phone Assessment
  • How It Works step 2: MATCHING PROCESS→Matching Process
  • How It Works step 3: WEEKLY CALLS→Weekly Calls
  • FAQ section H2: FREQUENTLY ASKED QUESTIONS→Frequently Asked Questions
  • Get Connected H2: READY TO MAKE A NEW FRIEND?→Ready to Make a New Friend?
  • Card 1: I WANT A COMPANION→I Want a Companion
  • Card 2: I WANT TO REFER SOMEONE→I Want to Refer Someone

Headlines to Leave Uppercase (For Now)

  • Buttons (GIVE, SUBSCRIBE, REFER AN OLDER ADULT, JOIN THE PROGRAM) — short labels, fine as-is
  • Footer column headings (Programs & Services, Get Involved, Support Our Mission)
  • Top nav items — these are existing site-wide style, not part of Paul's request

2. Remove Albiol Script Font from Typed Text

Albiol stays alive only inside images and graphic assets. Anywhere it appears as live text, swap it for the display font.

Spots to Change

  • "Get Connected" eyebrow above "Ready to Make a New Friend?" → swap to Roboto Condensed, magenta, smaller size with letter-spacing as an eyebrow label
  • "How It Works" script title → use the display heading font (Roboto Condensed, magenta) — keep the thin underline accent if you want a graphic touch
  • "Volunteer With Us" script in the volunteer banner (if present on beta) → same treatment
  • Footer logo script text (if any) → replace with friends-co-logo-white.png image instead of typed Albiol
  • Anywhere else with font-family: 'Albiol' → grep the page in dev tools, find each widget, change to Roboto Condensed or Open Sans as appropriate

Where Albiol Can Stay

  • Inside FriendsCo_Pattern.jpg and other graphic assets — the script is baked into the image, that's fine
  • Image overlays and headers/banners that are designed in Photoshop/Illustrator and exported as PNGs

3. Background Behind the Woman Image

Pick a direction from the three mockup options, get Paul's sign-off, then update Elementor.

Steps

  • Review the three mockup options (Solid Orange, Oversized Ampersand, Flower on Cream) and share with Paul
  • Once approved, in Elementor open the section column → Background → remove the FriendsCo_Pattern.jpg overlay (or drop opacity to under 5% if you want a hint of texture)
  • Apply the chosen background: solid orange, oversized icon, or cream + icon combo
  • Test at desktop, tablet, mobile — confirm the oversized icon (if used) doesn't crowd the headline or buttons on narrow widths

4. Pre-Publish QA

Final pass before pushing the beta to live.

Checks

  • Spell-check all headings after Title Case conversion (auto-converters miss small words like "a", "the", "of")
  • Confirm heading hierarchy is still correct (H1 → H2 → H3) after font changes
  • Run Lighthouse / accessibility check — uppercase-by-CSS reads differently than uppercase-by-content in some screen readers
  • Test on iPhone (Safari) and Android (Chrome) — Paul flagged mobile works well, confirm changes don't break that
  • Confirm the new woman-on-phone image is loaded into the WordPress media library at the right size (web-optimized, ~1600px wide max)
  • Send beta link to Paul for final approval before pushing to live
